# Mission

DSA Trainer is a personal practice system for developers who want to master Data Structures & Algorithms through pattern recognition — not memorization.

## Purpose

DSA Trainer is a **structured learning application** — its premise is taken seriously. The enemy is not the algorithm; it is randomness. Most developers fail technical interviews not because they lack intelligence but because they practice without a system. They grind LeetCode in circles, recognize familiar problems but not the underlying patterns, and burn out before they ever build intuition.

This app exists to break that cycle. Every day has a pattern. Every problem reinforces it. Progress is visible. The system replaces anxiety with deliberate repetition.

The learning lives in the patterns, not the problem count. The code itself is a genuine, well-structured TypeScript web application intended to show developers how to build a modern local-first tool with Next.js, Zustand, and a clean component architecture.

## Primary Audience

**Experienced developers preparing for interviews or sharpening problem-solving skills.** The codebase should be clear, idiomatic, and worth studying. Every architectural decision should be defensible as a teaching example. The UI should feel fast and frictionless — no login, no setup, no noise.

## Stakeholder Voices

- **The developer (engineering):** A reliable, fast local app. Clean architecture that scales from MVP to AI-assisted coaching without a rewrite.
- **The learner (product):** A 30-day roadmap that actually works. Daily pattern focus, problem tracking, and notes that persist without cloud friction.
- **The interviewer (outcome):** A candidate who recognizes patterns under pressure, not one who memorized solutions they can't explain.

## What Success Looks Like

A developer opening this app at day one should close it at day thirty having internalized the core patterns of algorithmic problem-solving — sliding window, two pointers, binary search, BFS/DFS, dynamic programming — and be able to recognize which tool to reach for before writing a single line of code.

A developer reading this codebase should understand how to build a local-first Next.js application with persistent client state, a modular component system, and a data layer that is ready to graduate from localStorage to a real backend when the time comes.
